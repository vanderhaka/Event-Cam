# Alignment Recovery Playbook

> What to do when alignment breaks and execution drifts off the chosen path.

---

## Trigger Conditions

Activate this playbook when any of the following occurs:

1. Alignment score drops below 13 for one week.
2. Priority 0 decisions are reopened unexpectedly.
3. Contradictory pricing/narrative appears in active materials.
4. Two consecutive weeks pass with no clear scale/hold/kill outcomes.
5. Leadership identifies path drift in monthly review.

---

## Recovery Levels

## Level 1 — Early Drift (24–72h response)

Symptoms:
- Yellow signal trend,
- minor doc inconsistency,
- delayed decision closure.

Actions:
1. Run focused decision review.
2. Patch documentation inconsistencies.
3. Assign owners for unresolved items.
4. Reconfirm weekly commitments.

Success criteria:
- all urgent inconsistencies resolved,
- owner/due dates assigned.

---

## Level 2 — Material Drift (72h–7d response)

Symptoms:
- score <13,
- unresolved Priority 0 dependencies,
- experiment overload,
- KPI ownership gaps.

Actions:
1. Freeze new initiatives for 1 week.
2. Run Priority 0 decision refresh.
3. Cut experiment load to <=5 with explicit closeouts.
4. Reissue single-path statement and boundaries.
5. Publish corrected weekly dashboard.

Success criteria:
- score recovers to >=13,
- no unresolved critical blockers.

---

## Level 3 — Critical Misalignment (Immediate reset)

Symptoms:
- score <9,
- conflicting strategic directions active,
- repeated leadership reversals without audit trail.

Actions:
1. Pause expansion activity immediately.
2. Run 90-minute alignment workshop.
3. Rebuild path worksheet from current reality.
4. Re-sign ownership (RACI) and path commitments.
5. Issue all-hands alignment reset communication.

Success criteria:
- one path re-established,
- Priority 0 closed,
- weekly cadence restarted with clean baseline.

---

## Recovery Sprint Checklist

- [ ] Activate blockers log with updated severities.
- [ ] Log all recovery decisions in decisions log.
- [ ] Update changelog with recovery actions.
- [ ] Refresh scorecard at end of recovery sprint.
- [ ] Confirm leadership sign-off before unfreezing growth initiatives.

---

## Recovery Exit Gate

Do not declare recovery complete until all are true:

- [ ] Alignment score >=17 for at least one cycle.
- [ ] No critical blockers open.
- [ ] Path boundaries are explicit and being enforced.
- [ ] Dashboard and meeting cadence have resumed normally.
